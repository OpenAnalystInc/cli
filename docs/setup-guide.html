02

⚙ Configure API Credentials

Set credentials for any provider. OpenAnalyst is the default — others activate automatically when their API key is set.

  ~/.zshrc
# ── OpenAnalyst (default) ── export OPENANALYST_AUTH_TOKEN="your-openanalyst-key" # ── Or any other provider ── export ANTHROPIC_API_KEY="sk-ant-..." # Claude export OPENAI_API_KEY="sk-..." # GPT / Codex export OPENROUTER_API_KEY="sk-or-..." # OpenRouter export XAI_API_KEY="xai-..." # Grok export BEDROCK_API_KEY="..." # Bedrock # ── Override default model (optional) ── export OPENANALYST_MODEL="openanalyst-beta"
PowerShell
# OpenAnalyst (default) $env:OPENANALYST_AUTH_TOKEN = "your-openanalyst-key" # Or any other provider $env:ANTHROPIC_API_KEY = "sk-ant-..." $env:OPENAI_API_KEY = "sk-..." $env:OPENROUTER_API_KEY = "sk-or-..." # Override default model (optional) $env:OPENANALYST_MODEL = "openanalyst-beta"

? FAQ

How do I switch between providers?

Set the API key for the provider you want, then either use --model flag or set OPENANALYST_MODEL env var. The CLI auto-detects the provider from the model name. For example, --model gpt-4o routes to OpenAI, --model sonnet routes to Anthropic/OpenAnalyst.

Can I use multiple providers at the same time?

Yes. Set all the API keys you have. The default model uses OpenAnalyst, but you can switch per-session using /model gpt-4o or start with openanalyst --model grok.

Where is the GitHub PAT needed?

Only for installation — the repo is private, so the install script needs a GitHub token to download the binary or clone the source. Generate one at github.com/settings/tokens with repo scope. The PAT is NOT sent to any LLM provider.

What does openanalyst init do?

Creates project configuration files: .openanalyst/ directory, .openanalyst.json settings, and OPENANALYST.md — a guidance file that gives the AI context about your project.

What slash commands are available?

/help · /status · /model · /cost · /compact · /clear · /config · /memory · /diff · /export · /session · /version · /permissions

Is my data sent to OpenAnalyst?

Only when you use the OpenAnalyst provider. If you set OPENAI_API_KEY and use --model gpt-4o, your data goes directly to OpenAI. Each provider connection is isolated. The CLI routes to whichever provider you select.

How do I update to a new version?

Re-run the install command. It will download the latest release binary and replace the existing one. Your configuration and sessions are preserved.
