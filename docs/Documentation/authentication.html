Authentication

Connect to any LLM provider using interactive login, API keys, or environment variables.

Authentication Methods

🔑

Interactive Login

Use the built-in picker flow to choose a provider, paste a key, and optionally set a default model.

🔐

API Key

Set environment variables or edit .env file directly.

📄

Credentials File

Saved automatically by openanalyst login to ~/.openanalyst/credentials.json.

Interactive Login

The recommended way to authenticate. OpenAnalyst, Anthropic, OpenAI, xAI, OpenRouter, Bedrock, and Stability use interactive API-key login. Google Gemini supports either browser OAuth or API key login.

Current Login Options
Most providers use interactive API-key login. Gemini also offers browser OAuth for users who prefer signing in with Google. 
# Interactive provider picker
openanalyst login

# Login to a specific provider
openanalyst login --provider anthropic

# Check logged-in providers
openanalyst whoami

How Interactive Login Works

  1. You run openanalyst login and pick a provider
  2. The CLI opens the right interactive flow for that provider
  3. For Gemini, you can choose browser OAuth or paste a GEMINI_API_KEY
  4. For other providers, you paste the provider key directly into the picker flow
  5. Credentials are stored in ~/.openanalyst/credentials.json and mirrored into ~/.openanalyst/.env
Security
Gemini browser OAuth uses PKCE (Proof Key for Code Exchange). API-key providers stay local to your OpenAnalyst config files unless you explicitly export them elsewhere.

API Key Authentication

Set API keys via environment variables or the .env file:

~/.openanalyst/.env
# OpenAnalyst (default provider)
OPENANALYST_API_KEY=your-openanalyst-key
OPENANALYST_BASE_URL=https://api.openanalyst.com/api

# Anthropic / Claude
ANTHROPIC_API_KEY=sk-ant-api03-...

# OpenAI / GPT / Codex
OPENAI_API_KEY=sk-...

# Google Gemini (also supports OAuth: openanalyst login)
GEMINI_API_KEY=AIzaSy...

# xAI / Grok
XAI_API_KEY=xai-...

# OpenRouter (350+ models)
OPENROUTER_API_KEY=sk-or-v1-...

# Amazon Bedrock
BEDROCK_API_KEY=your-bedrock-key

When you specify a model, OpenAnalyst resolves the provider and auth in this order:

Credentials File

After openanalyst login, credentials are saved to:

~/.openanalyst/credentials.json

This file can contain provider API keys, auth tokens, and provider metadata. It is automatically managed — you should not edit it manually.

Warning
Never commit credentials.json or .env to version control. The ~/.openanalyst/ directory should remain private.

Logging Out

# Logout from all providers
openanalyst logout

# Or from within the TUI
/logout
